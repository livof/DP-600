🧠 Semantic Models

DirectLake, storage modes, XMLA, relationships — the engine behind Power BI.

Core concepts first

⚡
DirectLake = the best of both worlds

DirectLake reads Delta files directly from OneLake — no data import, no DirectQuery latency. It's the magic mode that makes Fabric special. It falls back to DirectQuery if a file isn't framed yet. The exam loves asking when DirectLake falls back and what triggers it.

📦
Large semantic model storage = needed for XMLA writes

To use the XMLA endpoint for write operations (deploying models, scripting changes), the workspace must have Large semantic model storage format enabled. Small format = read-only XMLA. This is a common trap question.

🗂️
Star schema: dimensions surround facts

A fact table holds measurements (sales amount, quantity). Dimension tables hold context (dates, customers, products). They join on surrogate keys. Always set surrogate key columns' "Summarize By" to None — you never want Power BI summing a key column by accident.

🔀
cache.abf is the data file — keep it out of Git

When you save a Power BI project (.pbip), the semantic model folder contains a cache.abf file which holds cached data. Add it to .gitignore — you don't want actual data values committed to a repository.
