Concepts - read these first

🏅
Medallion = Bronze then Silver then Gold

Bronze is raw data, never delete it. Silver is cleaned & standardised. Gold is business-ready aggregates. This pattern preserves lineage & lets you reprocess at any layer without losing the original source data.

📋
Table.Profile() gives you instant column statistics

In Power Query (M language), Table.Profile() returns count, average, min, max, null count, & distinct count for every column in one call. Do not confuse it with Table.Schema() which only gives column types, not statistics.

⚡
V-Order makes Delta reads faster - applied at write time

V-Order is a Fabric write-time optimisation that compresses & sorts Delta table files so Power BI & Spark can read them faster. It is on by default. The exam tests whether you know it speeds up reads, even though it is applied during writes.

🏗️
Warehouse uses T-SQL. Lakehouse uses Spark & Delta.

The Fabric Data Warehouse is a traditional SQL engine: T-SQL, views, stored procedures. The Lakehouse runs Spark on Delta files. Structured SQL analytics goes to the Warehouse. Big data transforms go to the Lakehouse.

🔍
df.summary() adds percentiles on top of df.describe()

describe() gives stats for numeric & string columns. summary() adds the 25th, 50th, & 75th percentile values. If a question asks for min, max, mean, & standard deviation for all column types - use summary().
