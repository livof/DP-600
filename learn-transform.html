🔄 Transform & Model

Spark notebooks, SQL warehouses, and shaping data for analytics.

Core concepts first

🏅
Medallion = Bronze → Silver → Gold

Bronze: raw data exactly as it arrived (never delete this). Silver: cleaned, validated, standardised. Gold: business-ready aggregates and models. This pattern keeps data lineage clear and lets you reprocess without losing the original.

⚡
V-Order = faster reads from Delta tables

V-Order is a Fabric-specific write-time optimisation that compresses and sorts Delta table files so Power BI and Spark can read them faster. It's enabled by default. The exam tests whether you know it speeds up reads, not writes.

📋
Table.Profile() gives you column stats instantly

In Power Query (M language), Table.Profile() returns count, average, min, max, null count, and distinct count for each column in one shot. It's the go-to for quick data profiling. Don't confuse it with Table.Schema(), which gives column types — not statistics.

🏗️
Fabric Warehouse = T-SQL, not Spark

The Fabric Data Warehouse is a traditional SQL engine — you write T-SQL, create views, stored procedures, and run cross-database queries. The Lakehouse is for Spark/Delta. Know which tool fits which workload: analytical SQL → Warehouse; big data transforms → Lakehouse + Notebooks.
