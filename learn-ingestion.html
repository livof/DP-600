⚙️ Data Ingestion

How data gets into Fabric — pipelines, dataflows, notebooks, and shortcuts.

Core concepts first

🔗
Shortcuts vs Copy — know the difference

A shortcut doesn't move data — it's a live pointer to data sitting elsewhere (ADLS, S3, another Lakehouse). A pipeline Copy Activity physically moves data. Use shortcuts when you want zero duplication and real-time access. Use copy when you need a snapshot in Fabric.

🌊
Dataflows Gen2 = Power Query in the cloud

Dataflows let you transform data with a no-code Power Query interface. The output lands in a Lakehouse or Warehouse. They're great for business users, but they don't scale as well as Spark for huge datasets. Know when to use them vs Notebooks.

⚡
Eventstream = real-time data on a conveyor belt

Eventstream captures live events (from Azure Event Hubs, Kafka, IoT Hub) and routes them to KQL Databases, Lakehouses, or other Eventstreams — with optional transformations in between. Think of it as a real-time pipeline.

📊
PySpark wins at scale — pandas doesn't

When a question mentions billions of rows, parallel processing, or minimising load time — the answer is PySpark. pandas loads everything into memory on one machine. PySpark distributes the work across a cluster. This distinction comes up constantly in the exam.
