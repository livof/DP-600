⚙️ Data Ingestion

⏱ ~10 min

How data gets into Fabric - and which tool to use when.

Progress
0 / 10

Concepts - read these first

🔗
Shortcut = live pointer, no data copy

A shortcut points to data in ADLS Gen2, S3, or another Lakehouse without moving it. A Copy Activity physically brings data into Fabric. Use shortcuts to avoid duplication and always have fresh data. Use copy when you need a local snapshot.

📊
PySpark scales to billions of rows; pandas does not

When a question says "one billion rows", "parallel processing", or "minimise load time" - the answer is PySpark. pandas loads everything into RAM on one machine. PySpark distributes the work across a cluster. This distinction appears in multiple exam questions.

🌊
Dataflows Gen2 = Power Query in the cloud

Dataflows let you transform data with a no-code Power Query interface. Results land in a Lakehouse or Warehouse. Great for moderate-sized data and business users. Not the right tool for billions of rows - use Notebooks for that.

⚡
Eventstream routes live events with optional mid-stream transforms

Eventstream captures data from Event Hubs, Kafka, or IoT Hub and routes it to KQL Databases, Lakehouses, or other Eventstreams in real time. You can filter and transform the stream before it lands anywhere.

🔢
Descriptive what happened. Diagnostic why. Predictive what will. Prescriptive what to do.

The exam asks you to classify analytics scenarios. Column 📊s showing historical sales = descriptive. Finding the root cause of a drop = diagnostic. ML models forecasting churn = predictive. Recommending an action = prescriptive.
