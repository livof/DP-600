Domain 2 — Prepare and serve data

Managed vs external tables — who owns the data?

If you specify a path, it's external. If you don't, it's managed. That one line changes everything.

~5 min read · 1 exam question · 3 quiz questions

01 — The core idea

Two types of tables. One question: who owns the data?

Managed = Fabric owns the house. External = you own the house, Fabric has the address.

🏠 Managed table

Fabric owns everything

  • Fabric controls data + metadata
  • Drop table = data deleted
  • Fully integrated with Power BI, SQL endpoint
  • Fabric's default preference
📍 External table

You own the data

  • Fabric controls metadata only
  • Drop table = metadata gone, data stays
  • Not fully visible to Power BI or SQL endpoint
  • Data lives at your chosen path

02 — The code that decides everything

One line makes it external

USING DELTA OPTIONS (path '...') = external. No path = managed.

This creates an EXTERNAL table %%sql
CREATE TABLE FactRetailSalesFY23 (
  CustomerID STRING,
  ProductID STRING,
  SalesAmount STRING,
  ....
) USING DELTA OPTIONS (path 'Files/FactRetailSalesFY23');

-- ↑ THIS LINE is everything. You gave a path → external.
This would create a MANAGED table %%sql
CREATE TABLE FactRetailSalesFY23 (
  CustomerID STRING,
  ProductID STRING,
  SalesAmount STRING
);

-- ↑ No path. Fabric decides where data lives → managed.
The exam trick

The code looks like a normal CREATE TABLE. You have to spot the USING DELTA OPTIONS (path ...) at the bottom. That single clause flips everything from managed to external.

04 — Pattern recognition

How to spot it instantly on the exam

Scan the code for one thing: is there a path?

No path, no OPTIONS → Managed table. Fabric default. Everything works.

USING DELTA OPTIONS (path '...') → External table. You own the data. Limited integration.

Path in the code → External table

No path → managed. Path → external. That's the whole test.

Trigger words for managed:

CREATE TABLE (clean) no OPTIONS Tables/ folder fully integrated

Trigger words for external:

USING DELTA OPTIONS (path) Files/ folder ABFS path metadata only

05 — Real exam question

The actual Yes/No statements

Read the code, then decide each statement. Tap Yes or No.

The code from the exam %%sql
CREATE TABLE FactRetailSalesFY23 (
  CustomerID STRING, ProductID STRING,
  CurrencyID STRING, SalesAmount STRING, ....
) USING DELTA OPTIONS (path 'Files/FactRetailSalesFY23');

INSERT INTO FactRetailSalesFY23
SELECT * FROM FactRetailSalesFY23_view;

The code creates a Managed table in Fabric Lakehouse using SparkSQL.

No. The USING DELTA OPTIONS (path '...') clause makes this an external table, not managed. Without that OPTIONS line, it would be managed.

Dropping FactRetailSalesFY23 will delete metadata only.

Yes. Since it's an external table, dropping it only removes the metadata (the table definition). The actual data files at Files/FactRetailSalesFY23 remain untouched.

Data residing in FactRetailSalesFY23 cannot be accessed by using Power BI.

Yes. External tables with data in the Files/ path (ABFS) are not visible to Power BI, the SQL Analytics endpoint, or Dataflow Gen2. Only managed tables in the Tables/ folder get full integration.
