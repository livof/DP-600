01 — The story

What's going on here?

Shop + warehouse. Move data without crashing the shop.

Your OLTP system is a busy shop — orders, customers, all day. Your Lakehouse is a warehouse for analysis.

Why not just query the shop?

It slows everything down. Customers wait. Transactions break. Bad idea.

Synapse Pipelines

05 — The deeper concept

ETL vs ELT

Fabric uses ELT. Load first, transform later.

ETL (old)ELT (Fabric)
OrderTransform → LoadLoad → Transform
SpeedSlowerFaster
FlexibilitySchema locked earlyReshape anytime
Copy Activity = the "L" in ELT

It loads raw data fast. You transform later with Spark or SQL. That's the modern way.

Q1: You need to ingest data from Azure SQL into a Fabric Lakehouse daily. What do you use?

A. Logic Apps with a SQL connector
B. BCP utility with a cron job
C. Data Pipeline with Copy Activity
D. Power BI dataflow refresh
Copy Activity in a Pipeline = Fabric-native, scheduled, reliable, large-scale. Logic Apps = workflows. BCP = manual. Power BI dataflows = lightweight transforms, not bulk ingestion.

Q2: What's the main advantage of ELT (the approach Copy Activity supports)?

A. Data is cleaned before entering the lakehouse
B. Raw data lands fast, you transform later
C. It removes the need for a pipeline
D. It auto-creates star schemas
ELT = Extract, Load, Transform. Copy Activity does the E and L. Fast. The T comes later with Spark or SQL. More flexible than cleaning upfront.

Q3: When should you use an OneLake shortcut instead of Copy Activity?

A. When you need to transform during ingestion
B. When you need a daily schedule
C. When you want to physically move data
D. When you want to reference data without copying it
Shortcuts = reference in place, no copy. Copy Activity = physically move data. Key exam distinction: shortcut references, pipeline copies.
