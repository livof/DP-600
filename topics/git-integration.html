Connect Fabric to Git — the exact sequence

Org → Project → Repo → Branch → Folder. Big to small. Always.

~3 min read · 3 quiz questions

01 — The story

What's going on here?

You're connecting a Fabric workspace to Git for version control.

Fabric lets you hook your workspace into Azure DevOps so your work is version-controlled — just like code.

The exam tests whether you know the exact order you click through when setting this up. It's always top → down. Big → small.

The analogy that sticks

It's like opening folders on a computer: Company → Team → Project → Version → File. You'd never start at the file and work backwards.

02 — The sequence

Five steps. This exact order.

Org → Project → Repo → Branch → Folder. Done.

1

Organization

Your Azure DevOps organization. The top level.

Think: "the company"
2

Project

A project inside that org. Groups related work.

Think: "the department"
3

Git repository

The specific repo that holds your Fabric items.

Think: "the codebase"
4

Branch

Which branch to sync with. Usually main or dev.

Think: "the version"
5

Git folder

The folder path inside the repo. Where items live.

Think: "the location"
Org → Project → Repo → Branch → Folder

Big → small. Always. Never random order.

03 — Why this matters

Three reasons to care

Version control, collaboration, and CI/CD in Fabric.

🔄

Version control

Track changes to your Fabric items. Roll back if something breaks.

👥

Collaboration

Multiple people work on the same workspace without overwriting each other.

🚀

CI/CD

Deploy changes from dev → test → prod using Git branches and pipelines.

Where does this live?

Git integration sits in the workspace settings and connects to the governance layer of Fabric. It's Domain 1 material — planning and managing your solution.

04 — Exam traps

These look right but aren't

The exam loves to sneak these in as options.

Repository URL

You don't paste a URL. You select from the hierarchy dropdown — Org, Project, Repo.

Personal Access Token

Not needed. Fabric uses Entra ID (Azure AD) login. No manual tokens.

Make stage public

Completely unrelated. This is about deployment pipelines, not Git integration.

05 — The bigger picture

How Git fits into Fabric

It's one piece of the governance puzzle.

Git integration is part of how Fabric handles governance and deployment. Your workspace syncs to a repo. Changes commit to Git. Then deployment pipelines push from dev → test → prod.

Exam pattern

If the question mentions "Git setup" or "connect workspace to source control" — follow the hierarchy. If it mentions "deploy to production" — that's deployment pipelines, a different topic.

Fabric uses your Microsoft Entra ID login — the same identity you use for Fabric itself. No manual tokens, no SSH keys, no URLs with credentials.
