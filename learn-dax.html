📊 DAX & Power BI

Measures, file types, dynamic formats, and performance tools.

Core concepts first

🧮
CALCULATE = the most important DAX function

CALCULATE evaluates an expression in a modified filter context. Almost every complex measure uses it. The pattern is: CALCULATE([Measure], Filter1, Filter2). Time intelligence functions like SAMEPERIODLASTYEAR are always wrapped in CALCULATE — they only produce a filter, they don't calculate anything on their own.

📁
PBIT vs PBIX vs PBIP — know every format

PBIX: the standard file, contains data + report. PBIT: a template — has structure and connections but no data. PBIP: Power BI project, folder-based for Git. PBIDS: just a data source connection file. The exam tests "no data + pre-built structure" = PBIT.

💱
Dynamic format strings = currency symbol that changes

When a measure should display in different currencies based on a user's selection, use a dynamic format string. It's a DAX expression that returns a format string (like "€#,0.00") rather than a fixed one. Field parameters are for switching which measure/column displays — not for formatting.

🔍
Model view vs Model Explorer — different tools, different jobs

Model view (the diagram) shows tables, columns, and relationships — great for spotting columns with wrong settings like "Summarize By". Model Explorer (the tree panel) shows the full semantic model hierarchy including measures and calculation groups. Performance Analyzer is for report-level query timing.
