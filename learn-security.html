Concepts - read these first

🏢
Workspace roles: Admin then Member then Contributor then Viewer

Admin has full control and can delete the workspace. Member can publish and share. Contributor can create and edit items but cannot share. Viewer is read-only. Exam questions always ask for the minimum role needed - always go with least privilege.

🔒
Dynamic RLS filters each user automatically using USERPRINCIPALNAME()

Static RLS creates one role per group. Dynamic RLS creates one role with a DAX filter like [Email] = USERPRINCIPALNAME(). It automatically filters to each logged-in user's own data. Much easier to manage at scale than static roles.

🏛️
Only a Fabric Admin can create a Domain

Domains group workspaces by department. Only a Fabric Admin can create one. A Domain Admin (assigned by the Fabric Admin) manages that domain but cannot create new ones. This hierarchy is a common exam trap.

🏷️
Sensitivity labels travel with the data. Endorsement is about quality.

Sensitivity labels (Conf🆔ential, Highly Conf🆔ential) control access and travel with exports to Excel, PDF, etc. Endorsement (Promoted or Certified) signals data trustworthiness. These are completely separate concepts - do not mix them up.

🆔
Workspace 🆔entity lets items authenticate without stored credentials

A workspace 🆔entity is a managed 🆔entity scoped to a Fabric workspace. It allows pipelines and notebooks to connect to Azure Storage or Key Vault without storing credentials anywhere. Create one and grant it the right RBAC role on the target resource.
